# SyntheticPeople

A reproducible synthetic person-record benchmark for matchify, generated by
`matchify/utils/synthetic_data_generation/person_faker.py`.

The combined CSV (`CombinedPeople.csv`) was produced with:

```bash
python -m matchify.utils.synthetic_data_generation.person_faker \
  --output datasets/SyntheticPeople/CombinedPeople.csv \
  --num_records 500 \
  --duplicate_percentage 0.2 \
  --close_match_percentage 0.2 \
  --close_nonmatch_percentage 0.1 \
  --seed 0
```

Each record has an `id` (unique), a `group_id` (records sharing a `group_id`
are duplicates of one another), and the five fields exercised by the
type-aware normalizers in `ERBaseModel`: `first_name`, `last_name`,
`birthdate`, `address`, `phone_number`.

Composition (500 records, seed=0):
- ~50% unique singletons
- ~20% exact duplicates of unique records
- ~20% close-matches (one perturbed character/field) of unique records
- ~10% close-non-matches (perturbed but corresponding to no other record)

Only bundled dataset that exercises the type-aware normalizers. The
real-world benchmarks (Amazon-Google, Abt-Buy, DBLP-ACM) all use
`type: "other"`.
